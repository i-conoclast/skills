# 디버깅 도우미

문제 상황을 분석하고 해결 방안을 제시합니다.

## 디버깅 워크플로우

### 1단계: 문제 파악
- 에러 메시지 분석
- 관련 로그 확인
- 재현 조건 파악

### 2단계: 원인 추적
- 스택 트레이스 분석
- 관련 코드 검토
- 상태/데이터 흐름 추적

### 3단계: 수정 및 검증
- 수정 코드 제안 (diff 형태)
- 관련 유닛 테스트 실행
- E2E 테스트로 전체 흐름 확인

## 사용자 입력

$ARGUMENTS

문제 상황을 설명해주세요:
- 에러 메시지
- 재현 방법
- 예상 동작 vs 실제 동작
